# Jojo 艺术画廊大模型接入说明

当前静态原型使用 `analyzeArt()` 生成模拟评价。正式接入时建议增加一个自己的后端接口，浏览器只上传作品和文字说明，API Key 保存在服务器环境变量里。

## 推荐接口

`POST /api/analyze-art`

请求：

```json
{
  "title": "彩色机器人",
  "date": "2026-06-13",
  "note": "机器人会帮我收玩具。",
  "imageBase64": "data:image/png;base64,..."
}
```

响应：

```json
{
  "summary": "画面有清晰主体，孩子正在表达机械结构和功能想象。",
  "strengths": ["主体明确", "色彩选择大胆", "有故事动机"],
  "questions": ["这个机器人最重要的按钮是什么？", "它下一步会去哪里？"],
  "parentComment": "先描述你看见的具体内容，再鼓励 Jojo 讲故事。"
}
```

## 分析原则

1. 不给孩子贴固定标签，例如“有天赋”或“不擅长”。
2. 重点描述可观察事实：线条、颜色、主体、空间、故事。
3. 评价要鼓励继续表达，而不是把作品变成考试。
4. 每次输出 1 到 3 个开放问题，帮助家长和孩子继续聊。
5. 可以记录创作阶段变化，但避免横向比较其他孩子。
