# 一个周末给孩子做 Jojo LAB：用 vibe coding 定制暑假英语学习小应用

> README 内容整理自本地 CYNADU 文章：http://localhost:3000/the-lib?section=libsec_b7a98061d201b23406d8a00323c24104

## 本地运行

```bash
node server.js
```

然后打开 http://127.0.0.1:4173/。

## 项目故事

> 摘要：这篇文章记录作者用一个周末为孩子做 Jojo LAB 的过程：把五十音复习、英语单词、自然拼读、英文歌解析、儿童画廊和奖励抽卡做成一个适配电脑、iPad 和手机的小应用。重点不是炫技，而是展示 vibe coding 如何把家庭教育目标、兴趣激励、词库定制和成长记录变成可持续迭代的亲子工具。

利用一个周末的时间，我为即将放假的孩子编写了一个属于他的小应用，使用的是 vibe coding 的方式。

这个应用其实做 vibe coding 已经很久了。我一直在忙着给 AIGC 的流程做属于自己的工作流，也从最早打开 vibe coding 的界面完全不知道该干什么，到现在有几乎做不完的工作和想不完的创意。所以，既然接下来孩子想在暑假期间冲刺一下英语学习，索性就以对孩子的学习能力和确定的学习目标，做了一个专属于他的小应用。

_Image placeholder: LIB-jojo-lab-vibe-coding-kids-english-app-M01_

这个应用一共有如下几个部分组成。

## Kana Quest 五十音

孩子小时候常去日本旅游，本来都记得所有的平假名和片假名。这个五十音的小游戏可以帮助他复习已经有点忘记了的五十音识读。

_Image placeholder: LIB-jojo-lab-vibe-coding-kids-english-app-M02_

## Word Camp 单词营

这是小程序的重头戏，它的目的就是帮助孩子背单词。在它的界面下，我设定了三种不同的方式（读英文、选中文和填空，还有一个我忘了是什么），分别针对如今马上要上大班的孩子。他能够看到一个英文单词，知道它的中文意思，这样就可以有效地阅读一些更高级的英文绘本。因为孩子的中文掌握量已经到了几千个字，所以这个界面对他来说应该完全没有阅读障碍。

_Image placeholder: LIB-jojo-lab-vibe-coding-kids-english-app-M03_

英文填空我也做了一些设置，比如每天每个单词缺失的字母从两到三个不等，至少要有一个元音，这样也可以帮助他在拼写方面有初步的认知。

在这个界面下，我在设定里还可以选择不同的词库，其中包括 KAT 的基础词库和进阶词库。这些词库都可以从网上找到现成的，我们作为家长可以设定他每天背单词的目标。

_Image placeholder: LIB-jojo-lab-vibe-coding-kids-english-app-M04_

在背单词的规则里面，我是这样设定的：如果一个单词连续答对了，在规定目标里，比如设定每天背 50 个单词，这 50 个单词就会以随机的形式出现在这个界面上。孩子只要连续答对三次，这个单词就算掌握了。如果他答对的次数还不够多，这个单词就还会循环再次出现。

而一个单词如果他答错了三次，这个单词就会变成一个紫色的重难点单词，还会反复出现，直到他再连续答对三次为止。这个单词会恢复成白色，再答对三次才会变成红色，也就是说已掌握。

左侧有一个面对这个词库的综合进度条和当天任务的进度条，可以看到这个进度条就显示了孩子对这些单词的掌握程度。

奖励机制是每记住 10 个单词（也就是当有 10 个小红点被点亮时），孩子就会获得一个小金星。当他连续获得 10 个小金星，就会获得一个大金星。这个大金星会在之后的奖励小游戏里面有用。

## Phonics Jam 自然拼读

这一点上我也是扒了几个著名的自然拼读教程，让 vibe coding 自己去研究这个自然拼读究竟以怎样的方式学习最好。我选了对孩子此刻已经有了一定自然拼读基础的进度下，对他比较有益的一种方法，设计了整个自然拼读进程，可以帮助他在自然拼读的过程中完全掌握基础的英语发音。

_Image placeholder: LIB-jojo-lab-vibe-coding-kids-english-app-M05_

## Song Notes 乐记

孩子最近在学架子鼓，也学会了很多好听的英文歌。这些英文歌有很多都是摇滚乐，所以歌词解读起来还是有点麻烦的。我恰好可以通过这样的一个软件调用大模型，对英文歌进行针对孩子的解释，逐字逐句地解释。

还有一点就是可以把英文歌里所有歌词出现的单词，通过大模型让它恢复成原型，然后再把这些原型统一地变成一套词库。这样孩子就可以每学一首英文歌，同时把这个词库放在我们的 Word Camp 里面进行深度的记忆和学习。从他的兴趣出发，从他日常每天背的单词出发，我们可以自定义这个词库，这样孩子的学习也是事半功倍。

_Image placeholder: LIB-jojo-lab-vibe-coding-kids-english-app-M06_

## Art Gallery 画廊

这个我还没有完全建设好。其实是希望把他从小时候开始创作的所有艺术作品，都能够通过扫描、拍摄的方式存到这个画廊里面。也调用大模型对这个图片进行再次裁切，并且镶上合适的相框。这点我回头会去做，这样我们就可以拥有了一个对他过往所有作品的回溯。

_Image placeholder: LIB-jojo-lab-vibe-coding-kids-english-app-M07_

## Card Cottage 集卡小屋

孩子最近有点迷恋钢铁侠，所以就做了一个 50 张卡片的抽卡小机制。我会在互联网上搜索一些图片（因为这个软件是非商用的，我没有考虑到那么多的版权问题），找到 50 张孩子一定会喜欢的非常帅气的钢铁侠图片，把它作为卡片的正面。

孩子在前面的 Word Camp 里面一旦获得了一颗大金星，就可以到这来兑换一张钢铁侠的卡片。他可以随机选择一张卡片，然后翻过来，这张卡片就永久保存在他的图片库里面了。

接下来我还准备把这些卡片都给他打印成一个实体卡片，这样他就可以拥有属于自己的钢铁侠卡片套装。这样的套装在网上我还真没搜到，买不到我就自己给他做一套。

_Image placeholder: LIB-jojo-lab-vibe-coding-kids-english-app-M08_

## 界面与设备适配

还要说一下，整个页面的设计我也采用了一个对孩子比较友好的设计，不光是在电脑上，也可以在 iPad 上友好地使用。不管是横屏和竖屏，我都做了相应的匹配，甚至在妈妈的手机上也可以临时拿出来使用。

其界面设计非常友好：

- 字体比较大。
- 整个色彩采用了对孩子眼睛非常柔和的一套配色。

虽然我们严格控制孩子的屏幕时间，但我觉得也可以开始让他了解一些智能设备的使用了。接下来，我打算再给他配一个彩色的电子墨水触摸屏，这样他就可以在尽量少伤害眼睛的情况下，完成这个假期的英语学习。

## 为什么自己做一个英语学习 App

为什么市面上有这么多学英语的软件，我还要自己给孩子 vibe coding 一个呢？我觉得主要有以下几点原因。

### AI 的世界无限可能

这算是给孩子的一份答卷，让他知道只要掌握了基本能力，知道自己明确的目标，就可以通过 vibe coding 编写有用、好玩的小程序。这本身就是父子相处和家庭教育的一部分，既是送给他的小礼物，也是一种认知观念的灌输，让他亲自有所体会。

### 词库的灵活性与实时性

市面上软件的词库通常是固定的，而我们可以根据他当下的需求来定制。

- 比如他正在学习的英文歌曲，或者正在看的动画片，我们可以利用大模型把其中的核心单词提取出来，转化成他的学习词库。
- 甚至包括我最近给他买的英文绘本，也可以生成对应的词库。

这些日常能随时随地使用的词库，才是真正记牢、不会忘的。

### 成长记录与留念

这个软件还可以作为他成长的记录。我准备把他这些年来创作的有趣艺术作品和画作都输入进去，作为孩子成长的一个留念。所以大家可以看到，现在即便是封面，也是我用 ChatGPT 帮他做的一张图片集合。这张复古的小卡通画面包含了这个软件所有的功能，还有他喜爱的钢铁侠小背影，以及他自己童年的一张照片。

### 自定义奖励机制

就连奖励机制也可以由家长自定义。根据孩子最近喜欢的 IP 形象或事物，我们可以在虚拟空间帮他实现收集工作，并随时更新成不同的产品系列。当然，这仅限家用的前提下，不涉及侵权和商用。

## 关于记忆曲线

再补充一点，关于“记忆曲线”这件事情，我在这个版本里面并没有把它纳入到孩子的学单词逻辑里面。

因为我觉得对于此刻的孩子，我们更需要的是让他对英语产生一定的兴趣，而不是反复陷入到对一些单词的枯燥记忆中。因此，能够找到孩子会使用到英语、会阅读到或者听到英语的场景，再将这些场景作为强化他英语学习的方式，而不是单纯地使用记忆曲线。

也许对孩子来说，这种方法更为友善，也不会让他丧失对英语学习的兴趣。

## 后续计划

目前这个程序还有很多不成熟的地方，我也在考虑要不要把它直接做成一个可以在 iPad 上面安装的 App。

接下来的这一周，我还会不断完善它的代码，让功能变得更加完善。也期待大家给我一些更好的建议。

我把这个程序的源代码发到 GitHub 上面，而部署则是放在自己公司服务器上。这样能够让孩子和妈妈在暑假去旅游的时候，也能远程地打开这个软件，并且会不断更新它的相应功能。目前我设定的接入模型是 MiniMax，因为我之前为 OpenClaw 和 Hermes 的订阅还没用完，索性就给孩子用吧。

如果需要我增加其他模型的支持，或者对它的功能还有其他方面的建议，请随时给我留言。

## 这篇资料适合怎么用

这篇资料适合作为 vibe coding 的家庭教育案例：它不是只展示一个界面，而是把孩子的兴趣、当前学习目标、家长可调节的词库、奖励机制和成长记录合在同一个小应用里。

对 CYNADU 内部来说，它可以作为“面向具体对象定制工具”的案例：先明确使用者、学习目标和实际场景，再用 AI 与代码把工作流做成持续可迭代的小产品。
